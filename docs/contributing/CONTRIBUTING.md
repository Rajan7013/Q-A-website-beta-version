## Development Workflow

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Q-A-website.git
   cd Q-A-website
   ```
3. **Create a Branch** for your feature:
   ```bash
   git checkout -b feature/amazing-feature
   ```

---

## Coding Standards

### Structure
- Keep Backend logic in `backend-unified/`.
- Keep Frontend components in `frontend/src/components/`.
- Keep Documentation in `docs/`.

### Commits
- Use descriptive commit messages.
- Format: `[Type] Description`
- Example: `[Frontend] Added new chat bubble design`
- Example: `[Backend] Fixed PDF parsing error`

---

## Testing

Before submitting a Pull Request (PR):

1. **Test Locally**: Ensure the app runs without crashing.
2. **Verify Auth**: Can you still log in?
3. **Verify Uploads**: Can you upload a document?
4. **Linting**: Check for console errors.

---

## Pull Request Process

1. Push your branch to GitHub.
2. Open a Pull Request against the `main` branch.
3. Describe your changes clearly.
4. Link any related issues.
5. Wait for review!

---

## Questions?

Open an [Issue](https://github.com/Rajan7013/Q-A-website/issues) for discussion provided you have checked the documentation first.
